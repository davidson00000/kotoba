# 欠陥ログ (Defect Log) — Kotoba

重大度区分: **S1** ブロッカー（システム使用不可）/ **S2** 重大（中核機能が動作しない）/ **S3** 軽微（回避策あり）/ **S4** 外観のみ。

| ID | 日付 | 重大度 | コンポーネント | 内容 | 報告者 | 担当者 | 状態 | 修正コミット / PR | クローズ日 |
|----|---|---|---|---|---|---|---|---|---|
| D-0001 | YYYY-MM-DD | S? | | | QA | Dev | Open / In Progress / Fixed / Closed / Won't Fix | | |

## ルール

- テスト失敗が発生したら必ず欠陥エントリを起票する（暗黙的なfailは許容しない）。
- S1/S2はFixed状態、または変更要求（Change Request）により正式に延期されない限り、スプリントレビューをブロックする。
- 「Won't Fix」とする場合はこのログに理由を1行で記載する。
