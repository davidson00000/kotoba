## 1. 目的 / ビジネスケース

Jarvisスタイル（口頭サマリ + 画面詳細）の音声アシスタントを、(a) 動作するAI/クラウドプロダクトを生み出す手段として、かつ (b) AI/クラウドのイニシアチブに対するPMBOKベースのプログラムマネジメント遂行能力をキャリアチェンジの証跡として示す手段として構築する。

## 2. 目標（SMART）

| # | 目標 | 指標 | 目標日 |
|---|---|---|---|
| O1 | 音声→LLM→音声+画面のエンドツーエンドループが動作 | エンドツーエンドのデモ動画 | MVP + 8週 |
| O2 | バックエンドのクラウドデプロイ | 公開エンドポイントが到達可能 | MVP + 8週 |
| O3 | 全PMBOK成果物を常時最新に維持 | 成果物スロット充足率100%、かつ該当スプリント内に更新されること | 継続 |
| O4 | GitHubリポジトリの監査可能性 | 全コミットが変更ログまたはスプリントと紐づく | 継続 |

## 3. スコープ内 (In Scope)

- 音声入力（STT）
- LLMオーケストレーション（Anthropic API等）
- 音声出力（TTS）+ 同期された画面上の詳細パネル
- **MVPスキルセット（3スキル）:** 天気、リマインダー/タイマー、ウェブ検索。根拠: 3つの異なる呼び出しパターン（外部API / ローカル状態 / LLMベースの検索）を確保してアーキテクチャ上の網羅性を得つつ、認証とコストのオーバーヘッドを最小化する。最終スコープと受け入れ基準はスコープ記述書で詳細化する。
- バックエンドのクラウドデプロイ
- PMBOK成果物一式

## 4. スコープ外 (Out of Scope — 初期設定)

- モバイルアプリのパッケージング
- マルチユーザー / 単一ユーザートークン以上の認証
- カスタム・ウェイクワードの学習
- ハードウェア（Raspberry Pi、スマートスピーカー連携）
- カスタムのファインチューニングモデル

3. 教訓文書が生成され、公開可能な形で仕上がっている。
