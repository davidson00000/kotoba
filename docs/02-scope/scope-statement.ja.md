# スコープ記述書 (Scope Statement) — Kotoba

## プロダクト・スコープ記述

音声駆動型のパーソナルアシスタント。ユーザーがリクエストを発話すると、システムは (a) TTSによる口頭サマリと (b) 画面上のリッチな詳細パネル の両方で応答する。中核パイプライン: STT → インテント解析 / LLMオーケストレーション → スキル実行 → 合成応答（音声 + UI）。

## プロダクト受け入れ基準

1. エンドツーエンド・レイテンシ（音声入力から音声出力開始まで）: スキル非使用のLLM質問でp50 ≤ 5秒。
2. 最低**3スキル**が安定動作（各スキル20ケースのテストセットで ≥ 90% 合格）。
3. デプロイ済みバックエンドが別マシンから到達可能。
4. 音声出力と画面出力が常に整合（口頭サマリと表示詳細の矛盾なし）。

## 成果物

| # | 成果物 | 受け入れ基準 |
|---|---|---|
| D1 | STTモジュール | 英語・日本語入力対応、10秒クリップを2秒未満で転記 |
| D2 | LLMオーケストレータ | スキルディスパッチ + 純粋LLM回答へのフォールバック |
| D3 | スキル3種（初期リスト — 例: カレンダー / 天気 / ウェブ検索、確定はTBD） | 各スキルがテストセットを通過 |
| D4 | TTS + UIパネル | 同期動作、構造化データのレンダリング |
| D5 | クラウドバックエンド | デプロイ済み、ログ + コストダッシュボード |
| D6 | PMBOK成果物一式 | 17成果物がすべて整備・最新 |

## 除外事項

- モバイルネイティブアプリ
- マルチテナント / マルチユーザー
- ウェイクワード（"Hey Kotoba"）— MVPではPTTを採用
- セッション間の長期記憶（MVPはステートレス）
- カスタムファインチューニング

## 制約

- 予算: UAT期の上限 ≤ $60/月（クラウド + API）
- 期間: 終結まで12週間
- チーム: 人間1人 + AIエージェント6体
- 技術: 既存のKosukeラップトップで開発可能であること

## 前提

- Anthropic / OpenAI APIが継続利用可能でコスト枠内に収まる。
- Browser ChatGPTおよびClaudeがステークホルダー代理として利用可能。
- 規制対象外（個人プロジェクト、エンドユーザーのデータ収集なし）。
