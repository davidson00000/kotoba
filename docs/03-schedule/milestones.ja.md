# スケジュール / マイルストーン — Kotoba

全12週間のプログラム。2週間スプリント。スプリントレビュー = 金曜日。レトロ + 次スプリント計画も同日に実施。

## マイルストーン

| ID | マイルストーン | 週 | 終了基準 |
|---|---|---|---|
| M1 | 立上げ (Initiating) 完了 | 1 | プロジェクト憲章 + ステークホルダー登録簿が承認済み |
| M2 | 計画 (Planning) ベースライン確定 | 3 | スコープ / WBS / スケジュール / リスク / コミュニケーション / 品質 / TDD がすべて v1.0 |
| M3 | MVP ローカル動作 | 6 | ラップトップ上でボイスループが動作、スキル1種が合格 |
| M4 | 3スキル全動作 | 8 | 3スキルがそれぞれ合格率 ≥ 90% |
| M5 | クラウドデプロイ | 10 | 公開エンドポイント到達可能 + コストダッシュボード稼働 |
| M6 | UAT完了 | 11 | Customer（Browser Claude）がUATレポートを承認 |
| M7 | 終結 (Closing) | 12 | 教訓 + 終結レポート + GitHubタグ `v1.0` |

## スプリント・カレンダー

| スプリント | 週 | フォーカス |
|---|---|---|
| S1 | 1–2 | 立上げ + 計画キックオフ |
| S2 | 3–4 | TDD + STT/TTSスケルトン |
| S3 | 5–6 | LLMオーケストレータ + スキル1 → MVPローカル |
| S4 | 7–8 | スキル2・3 + UIパネル |
| S5 | 9–10 | クラウドデプロイ + 堅牢化 |
| S6 | 11–12 | UAT + 終結 |

## クリティカルパス（概略）

Charter → TDD → LLMオーケストレータ → スキルディスパッチャ → スキル1 → クラウドデプロイ → UAT

オーケストレータまたはデプロイでの遅延は1対1で終着点に響く。バッファはスプリント5に確保。

## トラッキング

実績は週次ステータスレポート (`docs/08-execution/status-reports/`) に記録する。スケジュール差異が20%を超えた場合は変更要求を起票する。
