# 変更ログ (Change Log) — Kotoba

統合変更管理 (Integrated Change Control)。ベースライン（スコープ、スケジュール、予算、品質、設計）の変更はすべてここに記録する。

## プロセス

1. 起票者は下記のフォーマットで変更要求 (Change Request, CR) エントリを作成する。
2. TPMが影響評価（スコープ / スケジュール / コスト / 品質 / リスク）を行う。
3. スポンサーが承認 / 却下 / 保留を判断する。本プログラムではスポンサー = TPM = Kosukeであるが、判断は必ず理由付きで文書化する。
4. 承認された場合、影響を受ける成果物を更新し、CRを実施済み (Implemented) にマークする。

## CRエントリ・テンプレート

```
### CR-NNN — <タイトル>
- 起票日: YYYY-MM-DD
- 起票者: <ロール>
- 種別: スコープ / スケジュール / 予算 / 品質 / 設計 / その他
- 内容: <何を・なぜ変更するか>
- 影響:
  - スコープ: …
  - スケジュール: …
  - コスト: …
  - 品質: …
  - リスク: …
- 検討した代替案: …
- 判断: 承認 / 却下 / 保留
- 判断の根拠: …
- 判断日: YYYY-MM-DD
- 判断者: <ロール>
- 影響を受ける成果物: <一覧>
- 状態: Proposed / Approved / Implemented / Rejected
```
