# Change Log — Kotoba

Integrated Change Control. Every change to a baseline (Scope, Schedule, Budget, Quality, Design) is logged here.

## Process

1. Requester writes a Change Request (CR) entry below.
2. TPM reviews impact (scope / schedule / cost / quality / risk).
3. Sponsor approves / rejects / defers. For this program Sponsor = TPM = Kosuke; decision still written down with rationale.
4. If approved, affected artifacts are updated; CR marked Implemented.

## CR Entry Template

```
### CR-NNN — <title>
- Date raised: YYYY-MM-DD
- Raised by: <role>
- Type: Scope / Schedule / Budget / Quality / Design / Other
- Description: <what is being changed and why>
- Impact:
  - Scope: …
  - Schedule: …
  - Cost: …
  - Quality: …
  - Risk: …
- Alternatives considered: …
- Decision: Approved / Rejected / Deferred
- Decision rationale: …
- Decision date: YYYY-MM-DD
- Decided by: <role>
- Affected artifacts: <list>
- Status: Proposed / Approved / Implemented / Rejected
```
